Under the hood

  • Gemini Live engine — a single WebSocket relay bridges the browser to Gemini; the API key stays server-side, never in the browser.
  • Voice-activity detection + barge-in — tuned so one utterance maps to one turn; you can cut her off mid-sentence and she yields.
  • Live two-way transcript — streamed and DOM-trimmed so long talks stay light.
  • 8 selectable voices — Aoede, Kore, Leda, Zephyr, Charon, Fenrir, Orus, Puck.
  • Silent web grounding — real-time Google Search for news, weather, prices, flights — woven into her answer, never announced.
  • Delivery control — "slow down," "be excited," "sound professional" change her tone instantly.

How a connection works

Connect→ OAuth consent→ Callback→ Encrypt & store→ Connected
  • Adapter-per-provider — Google, Notion, Todoist and Slack each live in one file behind a shared interface; a registry maps tools → provider. Adding one is a single new file.
  • Per-user & private — tokens are keyed by user, AES-256-GCM encrypted at rest in a nicole2_integrations table. The signed OAuth state binds each consent to the right user.
  • Key-gated — a provider with no server credentials simply shows "Coming soon." Nothing breaks.
  • Voice tool-calling, server-side — Nicole calls tools like book_meeting, draft_email, create_task, post_slack; they run on the server (where the tokens are) and she reads the result back in a breath.

Safety: irreversible actions — send email, post to Slack, book a meeting with guests — are confirmed before acting, enforced both in her reasoning and in code (confirmed:true gate). A mis-heard word can't auto-send. Reads and drafts run instantly.

How the coach works

  • A phase machine — each lesson runs intro → teach → model → practice → readiness → live rep → debrief, advancing as you engage. Scaffolding fades as you go (I-do → we-do → you-do).
  • Real frameworks — PIN (cold-call open), ASK (discovery), AER (price objection), STAR (interview), each with worked examples and key lines.
  • Live roleplay — a different voice plays the prospect and opens the scene; you practice, then get an engagement scorecard. An explicit turn indicator shows whose turn it is.
  • Custom & history — describe your own drill by voice or text and the AI builds it; past sessions and scorecards are saved.

Two density modes: during a live rep the screen is near-silent (aura + one cue) so you focus on talking; the rich scorecard and transcript appear only at the debrief.

How memory & control work

  • Persisted, not in the tab — facts live in a nicole2_memory Postgres table, UPSERT'd per (user, key); reloaded into her context at the start of every session.
  • Proactive & correctable — she saves your name, business, goals and preferences as she learns them; tell her to remember or forget anything.
  • Per-user — the live session runs as the signed-in user, verified by their token, so memory and connected accounts are private to each person.
  • Camera vision — turn on the camera, ask "what do you see?" and she describes the live frame in detail.
  • Voice UI control — "open training," "switch your voice," "mute yourself," "update my goals" — she operates the app and edits your profile by voice.
05 — Architecture

How it's built to stay up

A clean split: a React voice client, a Node relay that owns the Gemini connection and all secrets, and Postgres for durable state.

CLIENT

React + TypeScript

Voice capture, live transcript, animated avatar, the Talk / Training / Roleplay / Integrations screens. Holds no secrets.

RELAY

Node + WebSocket

One session per user bridges the browser to Gemini Live. Owns the API key, runs tool calls server-side, and orchestrates reconnect + summarization.

DATA

Postgres

Users, durable memory, training history, and encrypted integration tokens — each row scoped to its user.

STABILITY

Self-healing sessions

Auto-reconnect with seamless resume, terminal-failure detection (no retry storms), leak-safe teardown, never reconnects mid-sentence.

LONG RUNS

Live summarization

Older turns are compressed into a running summary (recent turns kept verbatim) before context can overflow — hour-long talks stay alive.

SAFETY

Confirm-before-act

Irreversible actions are gated in prompt and in code; OAuth tokens encrypted at rest; per-user isolation throughout.
